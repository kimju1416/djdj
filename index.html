STEP 02 — 수업 기획
어떤 버튼이 필요할까요?
만들기 전에 필요한 소리를 먼저 정리해요.
체크한 항목을 AI에게 그대로 전달하면 돼요.
📋 필요한 기능 체크하기 해당하는 것만 선택
⚠️
처음엔 5~8개로 시작하세요 버튼이 너무 많으면 수업 중에 헤매요. 핵심만 먼저 만들고 나중에 추가하는 게 훨씬 편해요.
💡 종목별 추천 버튼 예시
줄넘기

준비음악, 시작신호, 실패음, 기록갱신 효과음, 종료신호

농구

팀 등장곡(2팀), 3점슛, 파울, 타임아웃, 시작/종료

체육대회

반별 응원가, 종목시작, 1등/2등/3등 효과음, 시상식 음악

STEP 06 — 배포 준비
GitHub 가입 & 레포 만들기
GitHub는 파일을 올려두면 무료로 웹사이트를 만들어주는 서비스예요.
회원가입부터 차근차근 따라하세요.
① GitHub 회원가입
1
github.com 접속

크롬 브라우저 주소창에 github.com 입력 후 Enter

2
Sign up 클릭 → 이메일/비밀번호 입력

이메일은 평소 쓰는 것, 비밀번호는 8자리 이상

3
Username(아이디) 입력 ← 중요!

영문+숫자 조합으로 직접 정해요.
예: teacher-kim, pe-teacher2025
이 아이디가 나중에 사이트 주소에 들어가요 → 아이디.github.io/...

4
이메일로 온 인증 코드 입력

GitHub에서 이메일로 6자리 숫자 코드를 보내요.
메일함 확인 후 코드 입력하면 가입 완료!

② 레포지토리(저장소) 만들기
1
로그인 후 오른쪽 위 + 버튼 클릭

→ New repository 선택

2
Repository name 입력

예: baseball (영문 소문자, 공백 없이)
이 이름이 나중에 URL이 돼요

3
Public 선택 (필수!)

Private로 하면 무료로 사이트를 만들 수 없어요

4
Create repository 클릭

빈 저장소가 만들어졌어요!

💡
레포 이름이 URL이 돼요 내 아이디가 teacher123, 레포 이름이 baseball이면
완성된 주소는 teacher123.github.io/baseball 이에요
STEP 07 — 파일 업로드
파일 올리기
mp3 파일 용량에 따라 방법이 달라요.
파일 하나가 25MB 넘으면 GitHub Desktop을 써야 해요.
🔍
내 파일 용량 확인 방법 mp3 파일 선택 → 마우스 오른쪽 클릭 → 속성 → 파일 크기 확인
하나라도 25MB 초과면 → 아래 방법 2 사용
📦 방법 1 — 웹에서 직접 업로드 파일 전부 25MB 이하일 때
1
레포 페이지에서 Add file → Upload files 클릭
2
HTML 파일 먼저 올리기

파일을 점선 영역으로 드래그하거나 choose your files 클릭

3
Commit changes 클릭 (초록 버튼)

HTML 업로드 완료!

4
다시 Add file → Upload files 클릭

이번엔 music 폴더 안에 있는 mp3 파일들을 전체 선택해서 드래그해요.
파일이 올라가면 상단에 경로 입력창이 보여요.
거기에 파일명 앞에 music/ 를 추가해줘요.
예: player1.mp3 → music/player1.mp3

💡 더 쉬운 방법: 파일을 올리기 전에 상단 경로창에
레포이름/music/ 처럼 직접 입력하면 자동으로 music 폴더 안에 들어가요.

5
모든 mp3가 올라갔으면 Commit changes 클릭

파일이 많으면 시간이 걸릴 수 있어요. 기다려주세요.

6
레포 메인 페이지로 돌아가서 music 폴더가 생겼는지 확인

폴더 클릭해서 mp3 파일들이 다 있으면 업로드 완료!

💻 방법 2 — GitHub Desktop 사용 25MB 초과 파일 있을 때
1
GitHub Desktop 설치

desktop.github.com 접속 → Download for Windows 클릭 → 설치

2
Sign in to GitHub.com 클릭 → 로그인

방금 만든 GitHub 계정으로 로그인

3
File → Clone Repository → 내 레포 선택 → Clone

내 컴퓨터에 폴더가 생겨요

4
Show in Explorer 클릭 → 폴더 열기

생성된 폴더에 HTML 파일과 music 폴더를 통째로 복사해 넣기

5
GitHub Desktop으로 돌아오면 왼쪽에 파일 목록이 보임

아래 Summary 칸에 아무 글자나 입력 (예: 파일 추가)

6
Commit to main 클릭 → Push origin 클릭

업로드 완료! Push 버튼이 사라지면 성공이에요

✅
이 단계 완료 체크 GitHub 레포에 들어갔을 때 HTML 파일과 music 폴더가 모두 보이면 성공이에요.
STEP 08 — 배포 완료
GitHub Pages 켜기
설정 딱 한 번만 하면 링크가 생겨요.
이후로는 파일만 올리면 자동으로 업데이트돼요.
⚙️ Pages 설정 순서
1
레포 페이지 상단 Settings 탭 클릭

톱니바퀴 아이콘 옆에 있어요

2
왼쪽 메뉴에서 Pages 클릭

스크롤 내리면 Code and automation 섹션 안에 있어요

3
Source 아래 None → Deploy from a branch 선택
4
Branch: main 선택 → 폴더: / (root) 선택 → Save 클릭
5
1~3분 기다린 후 페이지 새로고침

상단에 링크가 생기면 완성! 🎉

🔇
소리가 안 나올 때 체크리스트 ① mp3 파일명 대소문자 확인 → player1.mp3 (소문자만)
② music 폴더가 HTML 파일과 같은 위치에 있는지 확인
③ 강력 새로고침 → Ctrl + Shift + R (윈도우) / Cmd + Shift + R (맥)
④ 파일 올린 후 1~3분 기다리기 (GitHub가 반영하는 시간)
🎊
완성!
